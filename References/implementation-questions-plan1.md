## Phase 1: MVP (4주)

### Week 1: 프로젝트 부트스트랩 및 병렬 개발 착수

#### 1.1 프로젝트 초기 설정

**환경 설정**
1. Next.js 14, TypeScript, Tailwind CSS, Shadcn/ui를 사용하는 프론트엔드 프로젝트를 생성해줘.
2. Python 3.11, FastAPI, Pydantic 2를 사용하는 백엔드 프로젝트를 생성해줘.
3. Docker와 docker-compose.yml 파일을 사용해서 개발 환경을 설정해줘.
4. 프로젝트 루트에 .gitignore 파일을 생성해줘. Python, Node.js, 환경변수 파일을 무시하도록 설정해줘.
5. 프로젝트의 README.md 파일을 작성해줘. 프로젝트 개요, 기술 스택, 설치 방법, 실행 방법을 포함해줘.

**프로젝트 구조**
6. 1번 안 문서의 "최종 프로젝트 구조"에 맞게 디렉토리 구조를 생성해줘.
7. .claude 디렉토리 구조를 만들어줘. subagents와 skills 디렉토리를 포함해줘.
8. templates 디렉토리를 만들고, claude-sonnet-4, gpt-4o, gemini-pro 하위 디렉토리를 생성해줘.

#### 1.2 claude.md 작성

**기본 구조**
9. claude.md 파일을 생성하고, CR 템플릿 허브 프로젝트의 목표를 작성해줘.
10. claude.md에 기술 스택 섹션을 추가해줘. 1번 안 문서의 표를 참고해서 작성해줘.
11. claude.md에 프로젝트 아키텍처 섹션을 추가하고, 핵심 5가지 원칙을 설명해줘.
12. claude.md에 프로젝트 구조를 트리 형태로 작성해줘.

**코드 표준**
13. claude.md에 Python 코드 표준 섹션을 추가해줘. PEP 8, 타입 힌트, docstring, 에러 처리 규칙을 포함해줘.
14. claude.md에 TypeScript 코드 표준 섹션을 추가해줘. ESLint, Prettier, 네이밍 컨벤션을 포함해줘.
15. claude.md에 Git 커밋 메시지 컨벤션을 추가해줘.
16. claude.md에 API 설계 원칙을 추가해줘. RESTful 설계, HTTP 상태 코드, 에러 응답 형식을 포함해줘.

**의사결정 로그**
17. claude.md에 의사결정 로그 섹션을 만들어줘. 날짜, 결정 사항, 이유를 기록할 수 있는 템플릿을 제공해줘.
18. 왜 Claude Haiku를 기사 분류에 사용하는지 의사결정 로그에 기록해줘.
19. 왜 MVP 단계에서 데이터베이스를 사용하지 않는지 의사결정 로그에 기록해줘.

#### 1.3 Sub-Agent 설정

**Article Scraper Agent**
20. `.claude/subagents/article-scraper.md` 파일을 생성해줘. 기사 스크래핑 전문 에이전트로 설정해줘.
21. article-scraper 에이전트의 역할과 책임을 정의해줘. 어떤 언론사를 지원하고, 어떤 데이터를 추출해야 하는지 명시해줘.
22. article-scraper 에이전트가 claude.md를 참조하도록 설정해줘.
23. article-scraper 에이전트의 출력 형식(JSON 스키마)을 정의해줘.

**API Classifier Agent**
24. `.claude/subagents/api-classifier.md` 파일을 생성해줘. 기사 분류 API 전문 에이전트로 설정해줘.
25. api-classifier 에이전트의 역할을 정의해줘. 어떤 기사 유형을 분류하는지 명시해줘.
26. api-classifier 에이전트가 사용할 AI 모델(Claude Haiku)과 API 엔드포인트를 정의해줘.
27. api-classifier 에이전트의 프롬프트 엔지니어링 전략을 작성해줘.

**Frontend UI Agent**
28. `.claude/subagents/frontend-ui.md` 파일을 생성해줘. UI/UX 개발 전문 에이전트로 설정해줘.
29. frontend-ui 에이전트가 Shadcn/ui 컴포넌트를 사용하도록 설정해줘.
30. frontend-ui 에이전트의 반응형 디자인 원칙을 정의해줘.
31. frontend-ui 에이전트가 구현해야 할 컴포넌트 목록을 작성해줘 (ArticleInputForm, ClassificationResult 등).

**Code Documenter Agent**
32. `.claude/subagents/code-documenter.md` 파일을 생성해줘. 문서화 및 템플릿 생성 전문 에이전트로 설정해줘.
33. code-documenter 에이전트의 문서 작성 스타일 가이드를 정의해줘.
34. code-documenter 에이전트가 CR 평가 기준을 참조하도록 설정해줘.

#### 1.4 병렬 개발 시작

**스크래핑 모듈**
35. 조선일보, 중앙일보, 한겨레 웹사이트의 HTML 구조를 분석해줘. 기사 제목, 본문, 작성자, 날짜를 추출하는 방법을 알려줘.
36. BeautifulSoup4를 사용해서 기사 스크래핑 모듈을 작성해줘. URL을 입력받아 기사 정보를 JSON으로 반환하는 함수를 만들어줘.
37. 스크래핑 모듈에 User-Agent 로테이션 기능을 추가해줘.
38. 스크래핑 실패 시 재시도 로직을 추가해줘. 최대 3번까지 재시도하도록 해줘.
39. 스크래핑 모듈에 타임아웃 설정을 추가해줘. 10초 이내에 응답이 없으면 에러를 발생시켜줘.
40. paywall이 있는 기사를 감지하고 에러 메시지를 반환하는 로직을 추가해줘.
41. 스크래핑 결과를 Pydantic 모델로 검증하는 코드를 작성해줘.

**분류 API**
42. Claude Haiku API를 사용해서 기사를 분류하는 함수를 작성해줘. 5가지 유형(스트레이트 뉴스, 해설, 인터뷰, 오피니언, 기타)으로 분류해줘.
43. 기사 분류 프롬프트를 작성해줘. Few-shot 예제를 포함해서 정확도를 높여줘.
44. 분류 결과에 신뢰도 점수를 포함하도록 해줘. 신뢰도가 낮으면 "기타"로 분류해줘.
45. FastAPI로 POST /api/classify 엔드포인트를 만들어줘. 기사 제목과 본문을 입력받아 분류 결과를 반환해줘.
46. Claude API 호출 실패 시 에러 처리를 추가해줘. 재시도 로직과 fallback 전략을 구현해줘.
47. API 키를 환경변수로 관리하도록 설정해줘. .env 파일을 사용해줘.

**프론트엔드 UI**
48. Shadcn/ui를 사용해서 ArticleInputForm 컴포넌트를 만들어줘. URL 입력 필드와 제출 버튼을 포함해줘.
49. URL 유효성 검증 로직을 추가해줘. 한국 언론사 URL인지 확인해줘.
50. ClassificationResult 컴포넌트를 만들어줘. 분류 결과와 신뢰도 점수를 표시해줘.
51. 로딩 상태를 표시하는 UI를 추가해줘. Spinner 컴포넌트를 사용해줘.
52. 에러 메시지를 표시하는 Toast 컴포넌트를 추가해줘.
53. Next.js App Router를 사용해서 페이지 라우팅을 설정해줘.

**분류 API 고도화**

**프론트엔드 UI 고도화**

#### 3.3 통합 테스트

104. 성능 테스트를 실행해줘. 응답 시간과 동시 사용자 처리 능력을 측정해줘.
