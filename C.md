# C안

## 🎯 최종 종합안: "품질 우선 + 사용자 존중" 설계

### 설계 원칙

```
1. 품질 최우선 (80%)
   └─ 최적 AI로 강력히 유도하되

2. 사용자 존중 (20%)
   └─ 최종 선택권은 보장

3. 투명한 교육
   └─ "왜"를 명확히 설명

4. 점진적 개선
   └─ 데이터 기반 지속 개선
```

## 🛠️ 구현 전략: Phase별 로드맵

### **Phase 0.5: 기술 검증 (1-2주)**

```
목표: 핵심 기능 실현 가능성 검증

□ 주요 5개 언론사 스크래핑 테스트
□ Claude API로 기사 유형 분류 정확도 측정
□ 템플릿 다운로드 메커니즘 검증
□ 기본 UI 프로토타입 (Figma)

산출물: 기술적 타당성 보고서
```

### **Phase 1.0: MVP (4-6주)** ← 친구A 제안 채택

```
목표: 핵심 플로우 작동

✅ 필수 기능
├─ 기사 URL 입력
├─ 규칙 기반 기사 유형 분류 (정확도 70% 목표)
├─ 최적 AI 추천 (단순 매칭 로직)
├─ 템플릿 다운로드 (2종: 최적 / 기본)
└─ 기본 마법사 UI

❌ 미포함
├─ 복잡한 성능 비교 시각화
├─ 게이미피케이션
├─ 커뮤니티 기능
└─ 사용자 계정

기술 스택:
- Frontend: Next.js + TailwindCSS
- Backend: Python FastAPI
- AI: Claude Haiku (저렴, 빠름)
- Storage: Vercel Blob Storage
- Deploy: Vercel (프론트+백 통합)

예상 비용: ~$50/월
```

### **Phase 1.3: 품질 강화 (3-4주)**

```
목표: 유도 시스템 구현

✅ 추가 기능
├─ LLM 기반 기사 분류 (정확도 90% 목표)
├─ 성능 비교 시각화 (Chart.js)
├─ 교육적 콘텐츠 페이지 (3-1)
├─ 차별화된 템플릿 (4-A / 4-B)
└─ 기사 유형 수정 기능 ← 친구C 제안

✅ A/B 테스트 시작
├─ 버튼 문구 테스트
├─ 경고 메시지 강도 테스트
└─ 전환율 측정

목표 KPI:
- 최적 AI 선택률 60% 이상
- 다운로드 완료율 80% 이상
```

### **Phase 1.5: 커뮤니티 & 데이터 (4-6주)**

```
목표: 피드백 루프 구축

✅ 추가 기능

├─ 평가 결과 업로드 시스템
├─ 기본 커뮤니티 (댓글, 좋아요)

- Auth: Clerk 또는 NextAuth

```

   ├─ 최적 AI 선택률: 75% 목표

   └─ 평균 평가 횟수/사용자: 3회 이상
